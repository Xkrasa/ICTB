5. gpt-image-2 size 选 1024x1024，但实际不是 1:1
当前代码层面已经修了一部分
现在前端 gpt-image-2 只显示 size，后端也会把 size 传给 edits 接口。
支持值：
auto
1024x1024
1536x1024
1024x1536
不建议采纳模型二的裁切方案
模型二说：
API 返回后裁切左侧第一块面板，强制变成 1:1
这个风险很大，不建议做。
原因：
- API 返回的不是“拼接原图”，而是模型生成结果。
- 如果强行按拼接图第一块裁切，可能把成品人物裁坏。
- 这属于后处理硬裁，不是接口参数对齐。
推荐修法
增加真实尺寸验证：
保存生成图前/后用 PIL 读取 width/height
写入 registry：
  width
  height
前端显示：
输出尺寸：1024 × 1024
这样可以判断到底是：
参数没生效
接口忽略 size
还是前端预览 object-fit 造成误判

8. 点击节点无法选中运行，必须框选

function runSelected() {

    runCanvas([...selectedNodes]);
